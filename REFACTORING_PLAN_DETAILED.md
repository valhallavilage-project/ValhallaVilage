# 🎯 ДЕТАЛЬНЫЙ ПЛАН РЕФАКТОРИНГА (файл за файлом)

## ФАЙЛ #1: ResourcesService.cs

### 📋 Что делает файл:
- **Основная функция:** Управление игровыми ресурсами (дерево, камень, золото и т.д.)
- **Методы:**
  - `GetSprite(ResourceId)` - получить иконку ресурса
  - `ChangeResource(ResourceId, int)` - изменить количество ресурса
  - `SetNewResourceValue()` - установить новое значение

### 🔍 Где используется:
- **При сборе ресурсов** (камни, деревья)
- **При открытии Inventory** (показать иконки)
- **При UI обновлении** (HUD с ресурсами)
- **При крафте/строительстве** (тратить ресурсы)

### 🐛 ПРОБЛЕМА В КОДЕ:

**Строка 77:**
```csharp
return _resourceSetConfig.items.First(x => id == new ResourceId(x.id))?.icon;
```

**Что не так:**
- `.First()` выбросит **InvalidOperationException** если ресурс с таким ID не найден в конфиге
- Это **крэшнет игру** если:
  - ID опечатан в коде
  - Конфиг не загрузился
  - ID ресурса удалён из конфига

### ✅ ИСПРАВЛЕНИЕ:

```csharp
public Sprite GetSprite(ResourceId id)
{
    var resourceConfig = _resourceSetConfig.items.FirstOrDefault(x => id == new ResourceId(x.id));
    if (resourceConfig == null)
    {
        Debug.LogWarning($"[ResourcesService] Resource config not found for ID: {id}");
        return null; // ← Вернёт null вместо крэша
    }
    return resourceConfig.icon;
}
```

### 🎮 ЧТО МОЖЕТ СЛОМАТЬСЯ:

**Теоретически:**
- Если какой-то UI ожидает что GetSprite() ВСЕГДА вернёт спрайт (не null)
- UI может показать пустую иконку вместо ресурса

**НО вероятность:** Очень низкая (5%)
- Unity UI корректно обрабатывает null спрайты (просто ничего не показывает)
- В коде уже есть `?.icon` (проверка на null)
